## 專案名稱：StockWinner AI 台股智能決策系統
**擔任職務**：前端工程師 / AI 應用開發者
**開發週期**：202X/XX - 202X/XX (自行填寫)

### 專案描述
開發一款基於 Generative AI 的台股即時分析應用程式，解決一般散戶難以解讀法人籌碼與盤面結構的痛點。系統整合 Google Gemini Pro/Flash 大型語言模型，模擬專業法人機構的決策邏輯，提供即時的「個股診斷」、「法人行為階段識別」與「動態交易策略」。

### 技術架構 (Tech Stack)
*   **核心框架**: React 19, TypeScript, Vite
*   **UI/UX**: Tailwind CSS (Glassmorphism 玻璃擬態風格), RWD 響應式設計
*   **AI 整合**: Google Gemini API (@google/genai), Prompt Engineering
*   **資料視覺化**: Recharts (客製化雷達圖、數據儀表板)
*   **狀態管理與優化**: LocalStorage Caching Strategy (減少 40% API 請求)

### 主要貢獻與成就 (Key Achievements)

**1. 建構 AI 驅動的金融分析引擎**
*   設計「法人行為狀態機 (State Machine)」，透過 Prompt Engineering 讓 AI 將複雜籌碼數據轉化為「佈局、試單、撤退」三種直觀的可視化階段。
*   實作「雙模式分析架構」，提供 Flash (快速掃描) 與 Pro (深度推理) 兩種模式，讓使用者能在效能與精確度之間取得平衡 (Trade-off)。

**2. 效能優化與使用者體驗**
*   開發 **智能緩存機制 (Smart Caching)**，將 AI 分析結果與市場榜單緩存於 LocalStorage (效期 15-30 分鐘)，顯著降低 API 成本並提升頁面載入速度至 <1秒。
*   採用 **Glassmorphism (玻璃擬態)** 設計語言，結合 Tailwind CSS 打造高質感的深色模式介面，提供媲美現代金融 App 的操作體驗。
*   解決 LLM 輸出不穩定問題，透過 **Structured Output (JSON Schema)** 強制規範 AI 輸出格式，確保前端渲染資料 (Types.ts) 的型別安全 (Type Safety)。

**3. 複雜資料視覺化**
*   利用 **Recharts** 實作多維度雷達圖，將「基本面、技術面、籌碼面、情緒面」等非結構化數據轉化為量化指標 (0-100分)。
*   整合外部新聞與社群數據，於前端即時呈現 AI 總結的市場情緒與風險預警。

### 解決難題 (Challenge & Solution)
*   **挑戰**: 大型語言模型 (LLM) 回應速度較慢，影響即時看盤體驗。
*   **解決**: 導入非同步處理與 Optimistic UI 設計，在 AI 思考期間展示動態骨架屏 (Skeleton) 與進度提示，並實作上述的緩存策略，大幅優化回訪體驗。
